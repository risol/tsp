# TSP 开发路线图

本文档列出 TSP (TypeScript Server Page) 的已完成功能、进行中的工作和未来计划。

## 📊 项目状态

- **当前版本**: 0.5.0
- **最后更新**: 2026-01-29
- **项目成熟度**: Beta

---

## ✅ 已完成功能

### 核心功能

#### 1. HTTP 服务器
- ✅ 基于 Deno 的 HTTP 服务器
- ✅ 可配置的监听端口（默认 9000）
- ✅ 可配置的文档根目录（默认 ./www）
- ✅ 开发/生产模式切换
- ✅ 优雅的错误处理

#### 2. TSX 页面执行
- ✅ 直接执行 `.tsx` 文件，无需预编译
- ✅ 基于 Preact 的 JSX 渲染
- ✅ 自动 URL 路由到文件路径映射
- ✅ 支持嵌套目录结构
- ✅ 支持 `index.tsx` 作为目录默认页面

#### 3. 请求处理
- ✅ 解析 HTTP 方法（GET/POST/PUT/PATCH/DELETE/HEAD/OPTIONS）
- ✅ URL 查询参数解析
- ✅ 请求体解析（JSON / form / text）
- ✅ Cookie 解析和管理
- ✅ 请求头访问

#### 4. 响应功能
- ✅ JSX → HTML 渲染
- ✅ HTTP 重定向支持
- ✅ 自定义 Response 对象
- ✅ 可配置的响应头（Cache-Control 等）
- ✅ 开发模式禁用缓存

#### 5. 智能缓存系统
- ✅ 基于文件修改时间（mtime）的缓存
- ✅ 自动缓存失效
- ✅ 内存缓存管理
- ✅ 生产模式预编译和缓存预热
- ✅ 开发模式按需编译

#### 6. 动态模块加载
- ✅ `deno run` 模式直接 import
- ✅ `deno compile` 模式使用 @deno/loader
- ✅ TSX → JavaScript 转译
- ✅ 混合加载策略（性能优化）

#### 7. 安全特性
- ✅ 路径穿越攻击防护（`../`）
- ✅ 路径归一化
- ✅ 文件类型白名单（仅允许 `.tsx`）
- ✅ 根目录边界检查
- ✅ URL 编码绕过防护

#### 8. 类型安全依赖注入
- ✅ 全局类型声明（`types.d.ts`）
- ✅ 类型安全的依赖注入系统
- ✅ 显式依赖声明（不依赖字符串解析）
- ✅ 依赖构建器（支持异步）
- ✅ 依赖注册/注销管理
- ✅ 全局 `Page` 函数（无需 import）

#### 9. 配置系统
- ✅ JSON 配置文件支持

## 📅 短期计划 (1-3 个月)

### 优先级：高

#### 1. 静态文件服务
**目标**: 支持 serving 静态资源（CSS、JS、图片等）

**功能**:
- [ ] 自动检测静态文件（.css, .js, .png, .jpg 等）
- [ ] MIME 类型自动识别
- [ ] 缓存头设置
- [ ] gzip/brotli 压缩
- [ ] ETag 支持

**实现方式**:
- 在 `router.ts` 中添加静态文件路由
- 优先级低于 TSX 文件（TSX 优先）
- 可配置的静态文件目录

#### 2. 中间件系统
**目标**: 实现中间件机制，允许在请求处理流程中插入自定义逻辑

**功能**:
- [ ] 中间件接口定义
- [ ] 中间件链执行
- [ ] 请求/响应拦截
- [ ] 中间件配置

**示例中间件**:
- 日志中间件（已有基础版本）
- 认证中间件
- CORS 中间件
- 限流中间件

**实现方式**:
```typescript
interface Middleware {
  (req: Request, ctx: PageContext): Promise<Response | void>;
}

// 使用
app.use(logger());
app.use(auth());
app.use(cors());
```

#### 3. WebSocket 支持
**目标**: 添加 WebSocket 支持，实现实时通信

**功能**:
- [ ] WebSocket 连接管理
- [ ] 消息广播
- [ ] 房间/命名空间
- [ ] 连接状态管理

**实现方式**:
- 基于 Deno 的 WebSocket API
- 在 `src/websocket.ts` 中实现
- 与页面系统集成

### 优先级：中

#### 4. Session 管理
**目标**: 内置 Session 管理功能

**功能**:
- [ ] Session 存储（内存/文件/Redis）
- [ ] Session ID 生成
- [ ] Cookie 管理
- [ ] Session 过期
- [ ] Flash 消息

**实现方式**:
```typescript
// 在 AppDeps 中内置 session 支持
declare global {
  interface AppDeps {
    session?: {
      get(key: string): Promise<unknown>;
      set(key: string, value: unknown): Promise<void>;
      destroy(): Promise<void>;
    };
  }
}
```

#### 5. 模板布局系统
**目标**: 支持布局（Layout）和模板继承

**功能**:
- [ ] 布局组件
- [ ] 内容插槽
- [ ] 嵌套布局
- [ ] 布局覆盖

**实现方式**:
```tsx
// layouts/Main.tsx
export function Layout({ children }) {
  return (
    <html>
      <head><title>My App</title></head>
      <body>
        <Header />
        {children}
        <Footer />
      </body>
    </html>
  );
}

// pages/index.tsx
export default async function(ctx) {
  return <Layout><h1>Home</h1></Layout>;
}
```

#### 6. 表单验证助手
**目标**: 提供表单验证工具和类型

**功能**:
- [ ] 验证器函数
- [ ] 类型定义生成
- [ ] 错误消息管理
- [ ] 与页面集成

**实现方式**:
```tsx

export default async function(ctx: PageContext) {
  const { id } = ctx.params; // 自动解析
  const user = await db.findUser(id);
  return { json: user }; // 自动序列化
}
```
