

我需要你用 **Deno + Eta** 实现一个“类 PHP-FPM”的模板执行引擎，并满足以下要求：

---

## 一、总体目标

实现一个可执行程序 `tsp-fpm`，行为类似 PHP：

* 常驻进程监听 HTTP 端口
* 从指定 `docroot` 目录读取 `.tsp/.jsp` 文件
* 每个文件本身就是「代码 + 模板」（不区分 views）
* 请求 URL 自动映射到文件并执行，输出 HTML

示例运行方式：

```bash
./tsp-fpm --root ./www --port 9000
```

---

## 二、技术栈

* Runtime：Deno
* 模板引擎：Eta
* 引擎本身使用 TypeScript 编写（无需 tsc，使用 Deno 原生 TS）
* 发布使用：`deno compile`

---

## 三、路由规则（文件映射）

URL → docroot 下文件：

```
/            → index.tsp | index.jsp
/a           → a.tsp | a/index.tsp
/a/b         → a/b.tsp
```

规则：

* 自动补全 `.tsp` / `.jsp`
* 支持目录默认页 `index.tsp`
* 防止路径穿越攻击（必须校验 realpath 在 root 内）
* 只允许访问 `.tsp/.jsp`

---

## 四、模板执行模型（类似 PHP）

`.tsp/.jsp` 本身就是 Eta 模板，包含：

* HTML
* `<% %>` JS 代码
* `<%= %>` 输出

模板通过变量 `it` 访问上下文：

```ts
it.method
it.query
it.body
it.cookies
it.headers
it.url
it.file
```

示例模板：

```eta
<h1>Hello <%= it.query.name ?? "world" %></h1>
<% if (it.method === "POST") { %>
  <pre><%= JSON.stringify(it.body) %></pre>
<% } %>
```

---

## 五、上下文对象（Template Context）

每个请求向模板注入：

```ts
type TemplateContext = {
  method: string
  url: URL
  headers: Headers
  query: Record<string,string>
  body: unknown
  cookies: Record<string,string>
  file: string
  root: string
}
```

## 六、性能要求（必须实现缓存）

实现模板编译缓存：

* 使用 `Map<filepath, { mtimeMs, renderFn }>`
* 每次请求只 stat 文件
* mtime 未变化 → 复用已编译模板
* 变更 → 重新编译

避免每次请求都重新 parse 模板
